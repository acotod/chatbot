Integración Facebook Login para Panel Administrativo

📋 Resumen General

Esta aplicación integra Facebook Login como método de autenticación exclusivo para administradores de la plataforma Zentra Bot. La app solicita únicamente los permisos email y public_profile, y no publica, modifica ni accede a otros recursos de Facebook.

✅ Permisos Solicitados

Permiso Propósito
email Obtener la dirección de correo electrónico del usuario para validar si existe una cuenta administrativa vinculada
public_profile Obtener el nombre e identificador de Facebook (para auditoría y registros de inicio de sesión)
ℹ️ Nota: No se solicitan permisos de páginas, eventos, publicaciones ni acceso a datos privados. La autenticación es de solo lectura.

🧪 Pasos para Probar la Integración

  1. Acceder a la pantalla de inicio de sesión del panel administrativo

    Abre el navegador e ingresa a la URL del panel administrativo. Busca el botón o enlace "Continuar con Facebook".

  2. Hacer clic en "Continuar con Facebook"

    Se abrirá el diálogo de autorización de Facebook. El navegador te redirigirá a https://www.facebook.com/login.php si no estás autenticado.

  3. Autorizar con una cuenta de prueba

    Inicia sesión o autoriza con una cuenta de Facebook de prueba que tenga:

    • ✓ Email disponible en el perfil
    • ✓ Email ya vinculado a una cuenta administrativa en la base de datos
  4. Verificar el resultado esperado

    Si la cuenta está vinculada, la app iniciará sesión correctamente. Se mostrarán los datos del usuario en el panel administrativo.

✨ Resultados Esperados

✅ Caso 1: Cuenta vinculada correctamente

  • El usuario es redirigido al panel administrativo después de autorizar
  • Se asignan tokens JWT válidos para la sesión
  • El email de Facebook coincide con el de la cuenta administrativa
  • Se registra la acción en auditoría con el método "facebook"

⚠️ Caso 2: Email no vinculado a administrador

  • El usuario intenta autorizar con un email que no está vinculado a ninguna cuenta administrativa
  • La app rechaza el acceso con HTTP 403
  • Se muestra el mensaje: No admin account is linked to this Facebook email
  • Se registra un intento fallido en auditoría

⚠️ Caso 3: Cuenta sin email

  • Si la cuenta de Facebook no tiene email disponible
  • La app rechaza con HTTP 400
  • Se muestra: Facebook account has no email available

⚙️ Flujo Técnico (Detalles para Revisores)

1. Frontend: Obtención del Token

La app carga el SDK de Facebook desde https://connect.facebook.net/es_LA/sdk.js y llama a FB.login() solicitando email,public_profile.

2. Backend: Validación y Autenticación

Endpoint: POST /auth/facebook

  • Recibe el accessToken en el body
  • Valida el token llamando a https://graph.facebook.com/v25.0/debug_token
  • Verifica que el app_id coincida con la configuración de la app
  • Obtiene el perfil: https://graph.facebook.com/v25.0/me?fields=id,email,name
  • Busca un administrador con ese email en la base de datos
  • Solo si existe, emite JWT y sesión

3. Auditoría

Cada login exitoso se registra con:

  • accion: "LOGIN"
  • metadata: { via: "facebook", facebookId: "..." }
  • IP del cliente y User-Agent

🛡️ Conformidad con Políticas de Meta

✓ La app cumple con:
  • ✅ Transparencia: Solo solicita permisos necesarios (email, public_profile)
  • ✅ Seguridad: Token validado en backend; no se expone el app secret en frontend
  • ✅ Datos del usuario: Solo se guardan email e identificador para autenticación; sin venta, transferencia ni uso secundario
  • ✅ Callback de eliminación: Implementado en POST /auth/facebook/data-deletion
  • ✅ No automatización de contenido: No publica, comenta, reacciona ni modifica contenido
  • ✅ No exfiltración de datos: No descarga listas de amigos, contactos ni datos privados

🗑️ Callback de Eliminación de Datos

URL pública: https://[tu-dominio]/auth/facebook/data-deletion

Cuando un usuario elimina la app desde Facebook, Meta envía una solicitud con un signed_request firmado. El backend:

  • Verifica la firma con el app secret
  • Extrae el Facebook ID
  • Busca y anonimiza datos vinculados (desvincula ID de Facebook, revoca sesiones)
  • Retorna un código de confirmación único a Meta

⚙️ Configuración Requerida

Variables de entorno necesarias:

FACEBOOK_APP_ID 4224350961162585 FACEBOOK_APP_SECRET [Configurar en servidor seguro] FACEBOOK_GRAPH_VERSION v25.0 (por defecto)
⚠️ Importante: El FACEBOOK_APP_SECRET nunca debe exponerse en el frontend ni en repositorios públicos. Solo debe estar configurado en variables de entorno del servidor.

🔬 Pruebas Unitarias

La suite de pruebas cubre:

  • ✅ Token de Facebook válido + email vinculado → login exitoso
  • ✅ Token válido + email no vinculado → rechazo HTTP 403
  • ✅ Token ausente → rechazo HTTP 400
  • ✅ Token inválido → rechazo HTTP 401
  • ✅ Cuenta bloqueada temporalmente → rechazo HTTP 423

❓ Preguntas Frecuentes

¿La app accede a las páginas de Facebook del usuario?

No. Solo se solicitan email y public_profile. No se solicitan permisos de página ni administración.

¿Qué pasa si el usuario cambia su email en Facebook?

La próxima vez que intente login, el nuevo email se utilizará para buscar la cuenta administrativa. Si el nuevo email no está vinculado, se rechazará el acceso.

¿Se publican datos en las redes del usuario?

No. La app es de solo lectura. No publica, comenta ni modifica contenido.

¿Cómo se eliminan los datos cuando el usuario revoca la app?

Meta envía una solicitud POST a /auth/facebook/data-deletion con el Facebook ID. El backend anonimiza los datos y retorna un código de confirmación único.
